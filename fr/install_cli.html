Déploiement de votre application avec l'interface de ligne de commande

Dernière mise à jour : 5 octobre 2016

Vous pouvez utiliser l'interface de ligne de commande pour déployer et modifier des applications et des instances de service.

Avant de commencer, installez les interfaces de ligne de commande IBM® Bluemix® et Cloud Foundry.

Restriction : les outils de ligne de commande ne sont pas pris en charge par Cygwin. Utilisez-les dans une fenêtre de ligne de commande autre que Cygwin.

Une fois les interfaces de ligne de commande installées, vous pouvez commencer :

  1. Téléchargez votre code de démarrage et procédez à l'extraction du package dans un nouveau répertoire afin de configurer votre environnement de développement.

  2. Placez-vous dans le répertoire dans lequel se trouve votre code.

    cd votre_nouveau_répertoire

  3. Modifiez le code de votre application si nécessaire. Il est recommandé d'exécuter l'application localement avant de la déployer dans IBM® Bluemix®.

    Remarquez le fichier manifest.yml. Lorsque vous déployez à nouveau votre application dans IBM® Bluemix®, il est utilisé pour déterminer l'adresse URL de votre application, l'allocation de mémoire, le nombre d'instance et d'autres paramètres essentiels. Vous pouvez en apprendre plus sur le fichier manifeste dans la documentation Cloud Foundry.

  4. Connectez-vous à IBM® Bluemix®.

    bluemix api https://api.nom_domaine

  5. Connectez-vous à Bluemix.

    bluemix login -u nom_utilisateur -o
nom_org -s nom_espace

    Si vous vous servez d'un ID fédéré, utilisez l'option -sso.

    bluemix login -u nom_utilisateur -o
"nom_organisation" -s
"nom_espace"
-sso

  6. Déployez votre application dans Bluemix. Pour plus d'informations sur la commande cf push, voir Téléchargement de votre application.

    cf push "nom_app"

  7. Accédez à votre application en entrant l'adresse URL suivante dans votre navigateur :

    hôte.nom_domaine_app
